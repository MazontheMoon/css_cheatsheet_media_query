What are Media Queries?

Media Queries are used to apply or change CSS styles for different media types, commonly for resizing elements for different viewport sizes but also for user preferences such as dark mode or no animation as well as for print view.

Media queries can obtain information from the browser such as:

  • width and height of viewport
  • width and height of device
  • orientation
  • resolution
  • user preferences (dark mode, no animation, etc.)

A media query has a media type and can contain one or more expressions:

@media screen and (min-width: 1024px) { 
    **styles go here**
}

Media Types and Operators

  • all
  • print
  • screen
  • speech
  • operators

all

Used for all media type devices (default)

@media all and (prefers-color-scheme: dark) {...}

print

Used for printers

@media print {…}

screen

Used for computer screens, tablets, smart-phones, etc.

@media screen and (orientation: landscape) {…}

speech

Used for screen readers

@media speech {…}

operators

Logical operators can be used with the media queries

  • not
  • and
  • only

not

inverts the meaning of the entire media query

and

combines a media feature with a media type or other media features

only

prevents older browsers that do not support media queries with media features from applying the specified styles

any-hover

Check if ANY available input can hover over elements

  • none
  • hover
@media (anyhover:none) {…} @media (anyhover:hover) {…}

any-pointer

Check if any available input is a pointing device and if so the accuracy

  • none
  • coarse
  • fine
@media (any-pointer: none){…} @media (any-pointer: coarse){…} @media (any-pointer: fine){…}

aspect-ratio

The ratio between width and height of viewport (also min-aspect-ratio and max-aspect-ratio).

@media (min-aspect-ratio: 8/5) {…} @media (max-aspect-ratio: 3/2) {…} @media (aspect-ratio: 1/1) {…}

If using min-aspect-ratio and/or max-aspect-ratio with aspect-ratio then aspect-ratio must be at the bottom to avoid overrides.

color

Check the number of bits per RGB colour component of the output device

@media (color) {…} /*any colour device*/ @media (min-color: 8) {…} /*any colour device with at least 8bit colour*/

color-gamut

Check the range of colours supported by end software or hardware

  • srgb
  • p3
  • rec2020
@media (color-gamut: srgb) {…} /*any colour device*/

color-index

Check number of entries in output device colour table (also min-color-index and max-color-index)

@media (color-index:256) {…} @media (min-color-index:15000) {…} @media (max-color-index:8000000) {…}

display-mode

Check display mode of application.

  • fullscreen
  • standalone
  • minimal-ui
  • browser
@media (display-mode:fullscreen) {…} /*all available display area used*/ @media (display-mode:standalone){…} /*will look and feel like a standalone app*/ @media (display-mode:minimal-ui) *will look and feel like a standalone app but only minimal nav elements*/ @media (display-mode:browser){…} /*opens in browser tab*/

dynamic-range

Check brightness, contrast and colour on end software/hardware

  • standard
  • high
@media (dynamic-range:standard) {…} @media (dynamic-range:high) {…}

forced-colors

Check if user had enabled forced colours mode

  • none
  • active
@media (dynamic-range:standard){…}

grid

Check if output device uses grid-based screen

@media (grid: 1) {…} @media (grid: 0) {…}

height

Used to apply styles based on viewport height (also min-height and >max-height @media (height: 360px) {…} @media (min-height: 250px) {…} @media (max-height: 800px) {…}

hover

Check if primary input can hover over elements

  • none
  • hover
@media (hover:hover) {…} @media (hover:none) {…}

inverted-colors

Check if user or OS is inverting colours

  • none
  • inverted
@media (inverted-colors:none) {…} @media (inverted-colors:inverted) {…}

monochrome

Check the number of bits per pixel in output device monochrome frame buffer (also min-monochrome and max-monochrome)

@media (monochrome:0) {…} /*no monochrome*/ @media (monochrome) {…} /*any monochrome device*/

orientation

Check orientation of viewport

  • landscape
  • portrait
@media (orientation: landscape) {…} @media (orientation: portrait) {…}

overflow-block

Check how the output device handles overflow along block axis

  • none
  • scroll
  • optional-paged
  • paged
@media (overflow-block: none) {…} /*overflowing content not displayed*/ @media (overflow-block: scroll) {…} /*overflowing content available by scrolling*/ @media (overflow-block: optional-paged) {…} /*overflowing content available by scrolling but page breaks can be manually triggered (see “break-inside”) to display content on the following page*/ @media (overflow-block: paged) {…} /*Overflowing content is displayed on following page*/

overflow-inline

Check how the output device handles overflow along inline axis

  • none
  • scroll
@media (overflow-inline: none) {…} /*overflowing content not displayed*/ @media (overflow-inline: scroll) {…} /*overflowing content available by scrolling*/

pointer

Check if user has pointing device (mouse) and how accurate it is

  • none
  • coarse
  • fine
@media (pointer: none) {…} @media (pointer: coarse) {…} @media (pointer: fine) {…}

prefers-color-scheme

Check if user preference set for colour mode

  • light
  • dark

prefers-contrast

Check if user preference set for contrast colour

  • no-preference
  • more
  • less
  • custom
@media (prefers-contrast: no-preference) {…} @media (prefers-contrast: more) {…} @media (prefers-contrast: less) {…} @media (prefers-contrast: custom) {…}

prefers-reduced-motion

Check if user preference set to minimize the amount of non-essential motion

  • no-preference
  • reduce
@media (prefers-reduced-motion: no-preference) {…} @media (prefers-reduced-motion: reduce) {…}

resolution

Check pixel density of output device (also min-resolution and max-resolution)

@media (resolution: 150dpi) {…} @media (min-resolution: 72dpi) {…} @media (max-resolution: 300dpi) {…}

scripting

Check if scripting (JS) available

  • none
  • initial-only
  • enabled
@media (scripting: none) {…} /*scripting unavailable*/ @media (scripting: initial-only) {…} @media (scripting: initial-only) {…} /*scripting available on initial page load but not afterwards/ @media (scripting: enabled) {…} /*scripting available and active*/

update

Check how frequently the output device is able to modify appearance of content

  • none/li>
  • slow
  • fast
code> /*once rendered the layout cannot be updated (eg. print document)*/ @media (update: slow) {…} *layout can dynamically change but not fast enough to be perceived as smooth animation (e.g. book readers)* @media (update: fast) {…} /*layout can dynamically change fast enough to be perceived as smooth animation (e.g. computer screens)/*

video-dynamic-range

Check the combination of brightness, contrast ratio, and color depth that are supported

  • standard
  • high
@media (video-dynamic-range: standard) {…}

Firefox ONLY!

width

Check width of viewport (also min-width and max-width)

@media (width: 765px) {…} @media (min-width: 320px) {…} @media (max-width: 1024px) {…}
